深度学习中的注意力机制综述

注意力机制自 2017 年 Transformer 架构提出以来，已成为现代深度学习模型的核心组件之一。本文从基础原理出发，系统梳理其在自然语言处理、计算机视觉等领域的演化路径……

与传统的循环神经网络相比，注意力机制能够并行处理序列数据，同时更好地捕捉长距离依赖关系。这一特性使其成为大规模预训练模型的首选架构……

这段关于视觉领域演化路径的表述值得展开，后面补一节对比图。

信息栏（Date/Author/Source Url 网格 + Tags chips）左右边界与下方正文标题、段落对齐，到 TOC 开始的位置为止 —— 不跨到 TOC 或笔记栏下面。 Source Url 是特意选的长且不含空格的值，用来展示 overflow-wrap: anywhere：换行到第二行，而不是溢出到右边 Tags 或下一列。 点击上方"向下滚动"按钮：信息栏和标题一起离开视口，只剩正文段落 —— 因为它是 <article class="prose-cn"> 内部的第一个元素， 和正文共享同一个滚动容器；TOC 因自身 position: sticky 保持可见，笔记栏固定宽度不受影响。

笔记锚点标记设计说明： mark.vdl-note-anchor 用暖灰色块（border-strong 低透明度）+ 点状下划线，与高亮 mark.vdl-hl 的黄/绿/红实色块、蓝色下划线在色相和纹理上都不同—— 刻意不引入紫色等新色相（DESIGN.md 明确「无紫色/渐变」的反 AI-slop 约束），联动强调态复用品牌唯一 accent（sage green），与 Tab 下划线、悬停锚点徽标等既有交互态保持同一套语言。 悬停正文锚点 ↔ 悬停笔记卡片会互相高亮（is-linked 类），点击正文锚点会让对应卡片滚动到可见并进入编辑态——对应实现里这几个状态由 hoveredNoteId / focusNoteId 两个提升到 tasks.$id.tsx 的共享状态驱动。
