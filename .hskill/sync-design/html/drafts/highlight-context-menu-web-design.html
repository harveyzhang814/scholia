高亮右键菜单 · 撤销入口

对应实现：web/src/components/reader.tsx 中的 ctxMenu 浮层（右键点击已存在的高亮 <mark> 弹出）。 目前只有「删除高亮」一项，新增「撤销」用于恢复最近一次高亮操作（创建 / 改色 / 删除），适用于文章与视频总结阅读页，两者共用同一个 Reader 组件。

交互演示
本次会话里，示例文中「这项结论在后续实验中被反复验证」这句红色高亮已被删除（模拟用户误删）。
右键点击（此处用「点击」模拟右键）任意一条现有高亮 → 弹出菜单 → 点击「撤销」→ 恢复被删除的那条高亮，并有短暂提示与描边动效反馈其位置。
撤销栈为空时，「撤销」变为置灰不可点状态。

示例：模型评估方法论

在对比不同评估方案时，我们发现离线指标与线上表现的相关性并不稳定， 尤其是在数据分布发生偏移的场景下。为此团队引入了额外的 滚动窗口回测机制，用于持续校准评估口径。

后续排查表明，部分特征存在轻微的未来信息泄漏， 这也是本次调整的直接诱因。

提示：点击任意高亮文本以模拟右键弹出菜单。

「撤销」恢复的是最近一次的全局高亮操作（创建 / 改色 / 删除中的任意一种），而不是绑定到当前点击的这条高亮 —— 因此点击任意一条高亮的菜单，「撤销」效果都相同。撤销后如果历史记录已空，菜单项自动变为置灰状态并提示「没有可撤销的高亮操作」。
